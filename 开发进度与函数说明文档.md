# 开发进度与函数说明文档

## 1. 项目概述

本项目是一个基于 Three.js 和 React 的 3D 游戏场景，主要实现了以下核心功能：
- 基于 Simplex 噪声的地形生成系统（以平原为主）
- 动态光线渲染系统（基于海拔差的四档光影算法）
- 角色物理系统（重力、碰撞检测、防止穿模）
- 场景元素渲染（草丛、靶子的自适应放置）
- 辅助光照系统（环境光、方向光、背光补光、色调映射）
- 后处理效果（Bloom 模糊效果）
- **射击系统框架（自由式和锁定式双模式）**
- **目标锁定机制（射线检测、多射线布局、动态补偿修正）**
- **子弹系统（对象池管理、方向计算、可视化）**

## 3. 核心函数说明

### 3.1 地形生成系统

#### 3.1.1 `generateTerrain(params: TerrainParams): TerrainData`

**功能**：生成基于 Simplex 噪声的地形数据，优化为以平原为主

**参数**：
- `width`：地形宽度
- `depth`：地形深度
- `segments`：地形分段数
- `seed`：随机种子（可选）
- `heightScale`：高度缩放因子
- `noiseScale`：噪声缩放因子
- `octaves`：噪声八度（可选）

**返回值**：包含地形高度数据和 `getHeightAt` 函数的对象

**实现细节**：
1. 使用 Simplex 噪声生成多八度地形
2. 应用平方函数和高度限制，使地形以平原为主（占50%左右）
3. 计算每个顶点的高度值
4. 提供 `getHeightAt` 函数用于查询任意坐标的地形高度

**使用场景**：在 `TerrainRenderer` 中生成地形几何体

### 3.1.2 `getHeightAt(x: number, z: number): number`

**功能**：获取指定坐标的地形高度

**参数**：
- `x`：X 坐标
- `z`：Z 坐标

**返回值**：地形高度值

**实现细节**：
1. 将世界坐标转换为地形网格坐标
2. 进行边界检查，防止越界
3. 使用双线性插值计算非整数坐标的地形高度

**使用场景**：
- 角色碰撞检测时获取地形高度
- 场景元素（草丛、靶子）放置时获取地形高度

### 3.2 光线渲染系统

#### 3.2.1 `updateTerrainColors()`

**功能**：更新地形顶点颜色，实现基于海拔差的四档光影效果

**参数**：无（使用组件内部的 `characterPosition` 状态）

**返回值**：无

**实现逻辑**：
1. 遍历地形所有顶点
2. 计算每个顶点与角色的高度差 `deltaY`
3. 根据 `deltaY` 计算基础亮度（四档亮度划分）
4. 计算顶点与角色的距离，对远处的暗部进行亮度调整
5. 对角色周围20格内的暗部区域提高亮度
6. 更新顶点颜色属性并标记为需要更新

**亮度划分**：
- 暗：1.0（比角色低很多，deltaY < -2.0）
- 较暗：3.0（比角色低一点，-2.0 ≤ deltaY < -0.5）或远处/角色周围的暗部
- 较亮：5.0（和角色差不多高，-0.5 ≤ deltaY < 2.0）
- 亮：5.8（比角色高很多，deltaY ≥ 2.0）

**使用场景**：
- 每帧调用更新地形颜色
- 角色位置变化时调用更新地形颜色

### 3.3 角色物理系统

#### 3.3.1 `applyGravityToCharacter(position: Position, velocity: Velocity, delta: number, getHeightAt?: (x: number, z: number) => number)`

**功能**：应用重力并处理角色与地形的碰撞检测

**参数**：
- `position`：角色当前位置
- `velocity`：角色当前速度
- `delta`：时间增量
- `getHeightAt`：获取地形高度的函数（可选）

**返回值**：包含新位置和新速度的对象

**实现逻辑**：
1. 应用重力，更新角色速度
2. 计算新的角色位置
3. 获取角色当前位置的地形高度
4. 计算角色脚的高度（地形高度 + 1.5 - 0.90）
5. 检测碰撞，确保角色脚不低于地形高度
6. 限制角色下落速度，防止穿模

**使用场景**：在 `MovementController` 中每帧调用

### 3.4 射击系统架构

#### 3.4.1 系统概述

射击系统采用**接口+抽象类+管理器**的架构设计，支持双模式动态切换：

```
IShootingSystem (接口)
    ↓
BaseShootingSystem (抽象类，提供通用实现)
    ↓
    ├── FreeStyleShootingSystem (自由式)
    └── LockonShootingSystem (锁定式)

ShootingSystemManager (管理器，统一调度)
```

#### 3.4.2 IShootingSystem 接口

**功能**：定义所有射击系统的标准接口

**接口方法**：
- `name`: 射击系统名称
- `initialize()`: 初始化射击系统
- `update(delta: number)`: 每帧更新
- `setActive(active: boolean)`: 设置是否激活
- `isActive(): boolean`: 获取激活状态
- `setCamera(camera: THREE.Camera)`: 设置相机引用
- `setCanvas(canvas: HTMLCanvasElement)`: 设置画布引用
- `setCharacterPosition(pos: { x, y, z })`: 设置角色位置
- `handleMouseDown(event: MouseEvent)`: 处理鼠标按下
- `handleMouseUp(event: MouseEvent)`: 处理鼠标释放
- `handleMouseMove(event: MouseEvent)`: 处理鼠标移动
- `dispose()`: 销毁射击系统
- `setCallbacks(callbacks: ShootingCallbacks)`: 设置回调函数

**使用场景**：`ShootingSystemManager` 通过此接口统一管理所有射击系统

#### 3.4.3 BaseShootingSystem 抽象类

**功能**：提供射击系统的通用实现

**通用属性**：
- `callbacks`: 回调函数集合
- `isActive`: 激活状态
- `bulletVelocity`: 子弹速度（默认50）
- `fireRate`: 射击间隔（默认200ms）
- `characterPosition`: 角色位置
- `camera`: 相机引用
- `canvas`: 画布引用

**通用方法**：
- `createBullet(origin, direction)`: 创建子弹，触发回调通知管理器

**使用场景**：所有射击系统继承此类扩展各自逻辑

#### 3.4.4 ShootingSystemManager 管理器

**功能**：统一管理射击系统的注册、切换和更新

**核心方法**：
- `registerSystem(system: IShootingSystem)`: 注册射击系统
- `switchSystem(name: string)`: 切换到指定射击系统
- `getActiveSystemName(): string`: 获取当前系统名称
- `getActiveSystem(): IShootingSystem`: 获取当前系统实例
- `update(delta: number)`: 更新当前系统
- `handleMouseDown(event: MouseEvent)`: 转发鼠标事件
- `handleMouseUp(event: MouseEvent)`: 转发鼠标事件
- `handleMouseMove(event: MouseEvent)`: 转发鼠标事件

**使用场景**：`App.tsx` 中管理射击系统生命周期

### 3.5 自由式射击系统

#### 3.5.1 FreeStyleShootingSystem

**功能**：自由式射击模式，鼠标锁定状态下从相机中心发射

**射击逻辑**：
1. 检测鼠标是否按下
2. 检测射击间隔（200ms）
3. 从相机位置发射射线检测击中点
4. 计算从角色头顶到击中点的方向
5. 创建子弹向击中点飞行

**射线检测**：
```typescript
private getRaycastHitPoint(): THREE.Vector3 | null {
  const raycaster = new THREE.Raycaster();
  const direction = new THREE.Vector3();
  this.camera.getWorldDirection(direction);
  raycaster.set(this.camera.position, direction);

  if (this.shootableObjects.length > 0) {
    const intersects = raycaster.intersectObjects(this.shootableObjects, true);
    if (intersects.length > 0) {
      return intersects[0].point;
    }
  }

  // 未击中任何物体时，返回射线最远点
  return this.camera.position.clone().add(direction.multiplyScalar(100));
}
```

**子弹方向计算**：
```typescript
const hitPoint = this.getRaycastHitPoint();
const bulletOrigin = new THREE.Vector3(
  this.characterPosition.x,
  this.characterPosition.y + 1.5,
  this.characterPosition.z
);
const direction = hitPoint.clone().sub(bulletOrigin).normalize();
```

**使用场景**：默认射击模式，按住鼠标左键持续射击

### 3.6 锁定式射击系统

#### 3.6.1 LockonShootingSystem

**功能**：锁定式射击模式，射线检测目标后进入锁定倒计时

**锁定流程**：
1. 持续发射5条射线（十字形布局）检测目标
2. 检测到目标后开始1秒倒计时
3. 倒计时期间目标被遮挡则取消锁定
4. 倒计时结束锁定成功，射击方向指向目标
5. 目标脱离视野则取消锁定

**多射线布局**：
```
    [1]
     |
[3]-[0]-[4]
     |
    [2]
```
- `[0]`: 中心射线（朝向相机方向）
- `[1]`: 上方射线
- `[2]`: 下方射线
- `[3]`: 左侧射线
- `[4]`: 右侧射线

**动态补偿修正**：
```typescript
export function getCorrectedNDC(
  canvas: HTMLCanvasElement,
  clientX: number,
  clientY: number,
  camera: THREE.Camera,
  characterPosition: THREE.Vector3,
  baseCompensation: number = 0.3
): CorrectedNDCResult {
  // 1. 计算基础 NDC 坐标
  const ndcX = ((clientX - rect.left) / rect.width) * 2 - 1;
  const ndcY = -((clientY - rect.top) / rect.height) * 2 + 1;

  // 2. 计算相机仰角
  const pitch = getCameraPitch(camera);

  // 3. 距离补偿：远距离（>25米）完全不修正
  const distanceToCamera = camera.position.distanceTo(characterPosition);
  const distanceCompensation = distanceToCamera > 25 ? 0 : Math.min(1.2, 20 / distanceToCamera);

  // 4. 俯角补偿：俯角越大，补偿系数越大
  const pitchCompensation = 1.0 + Math.max(0, pitch) * 0.5;

  // 5. 综合补偿系数
  const totalCompensation = baseCompensation * distanceCompensation * pitchCompensation;
  const correction = -pitch * totalCompensation;

  // 6. 应用修正
  const correctedNDC = new THREE.Vector2(ndcX, ndcY + correction);
  return { raw: rawNDC, corrected: correctedNDC };
}
```

**动态射线检测范围**：
- 俯角越大，射线检测的横向面积越大
- 基础偏移量：0.05
- 俯角影响：offsetScale = 0.05 + pitchAbs * 0.1

**使用场景**：按Tab切换到锁定模式，对固定目标进行精确打击

## 5. 性能优化

### 5.1 计算优化

- **按需计算**：只在必要时更新地形颜色
- **边界检查**：避免访问不存在的属性
- **高效循环**：使用简单的循环结构，避免复杂的计算
- **直接计算**：只计算海拔差和距离，不需要复杂的向量运算
- **对象池**：子弹使用对象池复用，避免频繁GC

### 5.2 渲染优化

- **顶点颜色**：使用顶点颜色而非复杂的光照计算，减少渲染开销
- **按需更新**：只在必要时更新颜色属性
- **GPU 加速**：顶点颜色计算在 GPU 上执行，效率高
- **无阴影**：辅助光照系统关闭阴影，减少渲染开销
- **地形细分度优化**：从64减少到32，减少顶点数量和计算量

### 5.3 资源优化

- **纹理重复**：使用纹理重复而非多个纹理，减少内存使用
- **高效加载**：使用 `useLoader` 钩子加载纹理，确保资源加载完成后再渲染

## 9. 常见问题与解决方案

### 9.1 性能问题

**问题**：对于非常大的地形，计算量可能过大

**解决方案**：
- 使用空间分区技术，只计算角色周围的地形
- 实现 LOD（Level of Detail），根据距离调整地形细分程度
- 减少每帧计算次数，例如每 2 帧计算一次
- 优化地形细分度，减少顶点数量

### 9.2 视觉问题

**问题**：地形明暗过渡不自然

**解决方案**：
- 调整亮度阈值，使过渡更加平滑
- 增加辅助光照强度，提供更柔和的过渡
- 使用后处理效果，如 bloom，增强视觉效果

**问题**：远处地形过暗

**解决方案**：
- 调整远处亮度调整距离
- 增加远处暗部的亮度系数
- 调整环境光强度

### 9.3 技术问题

**问题**：角色穿模

**解决方案**：
- 优化碰撞检测逻辑，确保角色脚位置不低于地形高度
- 限制角色下落速度，防止穿透地形
- 增加碰撞检测频率

**问题**：纹理加载失败

**解决方案**：
- 检查纹理路径是否正确
- 确保纹理文件存在
- 使用错误处理，当纹理加载失败时使用默认颜色

**问题**：Bloom 效果不生效

**解决方案**：
- 确保安装了 @react-three/postprocessing 依赖
- 正确配置 Bloom 组件的参数
- 调整亮度值，确保有足够亮的区域触发 Bloom 效果

### 9.4 射击系统问题

**问题**：自由式子弹偏离准心

**解决方案**：
- 使用射线检测获取实际击中点
- 计算从角色头顶到击中点的方向作为子弹方向
- 调整准心偏移量与射线检测匹配

**问题**：锁定式目标丢失

**解决方案**：
- 检查射线是否被遮挡
- 确保目标在相机视野范围内
- 调整锁定取消条件
