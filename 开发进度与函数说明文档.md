**核心函数**：

```tsx
const App = () => {

  };

      <Canvas camera={{ position: [0, 5, 10] }}>

```

- 集成SceneSetup组件设置场景
- 集成PaperCharacter组件渲染角色
- 定义handleCharacterClick函数处理角色点击事件

export const SceneSetup = () => {

**功能说明**：

### 3.3 角色系统

#### PaperCharacter.tsx

**文件路径**：`src/systems/character/PaperCharacter.tsx`  
**功能**：渲染纸片人角色，实现角色的交互  

**核心函数**：

```tsx
export const PaperCharacter = ({ characterId, onClick }: PaperCharacterProps) => {
  const meshRef = useRef<any>(null);
  
  // 使用 useLoader 加载纹理
  const texture = useLoader(TextureLoader, '/textures/character.png');

  return (
    <Billboard>
      <mesh 
        ref={meshRef} 
        position={[0, FIXED_Y, 0]}
        userData={{ characterId }}
        onPointerDown={() => onClick(characterId)}
      >
        <planeGeometry args={[2, 3]} />
        <meshBasicMaterial 
          map={texture} 
          transparent={true} 
          opacity={0.8} 
        />
      </mesh>
    </Billboard>
  );
};
```

**参数说明**：
- `characterId`：字符串，角色ID，用于标识角色
- `onClick`：函数，点击回调函数，当角色被点击时触发

**返回值**：React组件

**功能说明**：
- 使用useRef创建mesh引用
- 使用useLoader加载角色纹理
- 使用Billboard组件实现角色始终面向相机的效果
- 创建平面几何体作为角色的形状
- 设置材质，应用纹理并启用透明效果
- 处理角色点击事件，调用传入的onClick回调

### 3.4 状态管理系统

#### gameStore.ts

**文件路径**：`src/systems/state/gameStore.ts`  
**功能**：管理全局游戏状态  

**核心函数**：

```tsx
export const useGameStore = create<GameState>((set, get) => ({
  // 初始状态
  character: {
    position: { x: 0, y: 0, z: 0 },
    isMoving: false
  },
  dialog: {
    text: '',
    visible: false
  },
  isLoading: false,
  isMuted: false,
  volume: 1,
  playSoundCallback: null,

  // Actions
  setCharacterPosition: (pos) => set((state) => ({
    character: {
      ...state.character,
      position: { ...state.character.position, x: pos.x, z: pos.z }
    }
  })),
  setCharacterMoving: (moving) => set((state) => ({
    character: {
      ...state.character,
      isMoving: moving
    }
  })),
  setDialogText: (text) => set((state) => ({
    dialog: {
      ...state.dialog,
      text
    }
  })),
  setDialogVisible: (visible) => set((state) => ({
    dialog: {
      ...state.dialog,
      visible
    }
  })),
  setIsLoading: (loading) => set({ isLoading: loading }),
  toggleMute: () => set((state) => ({
    isMuted: !state.isMuted
  })),
  registerSoundCallback: (cb) => set({ playSoundCallback: cb }),
  playSound: () => {
    const callback = get().playSoundCallback;
    if (callback) {
      callback();
    }
  }
}));
```

**功能说明**：
- 使用Zustand创建游戏状态存储
- 定义初始状态，包括角色状态、对话状态、加载状态和音频状态
- 提供状态修改的方法，如设置角色位置、对话文本等
- 实现音效播放的回调注册和触发

#### storeTypes.ts

**文件路径**：`src/systems/state/storeTypes.ts`  
**功能**：定义状态管理的类型  

**核心类型**：

```tsx
export interface Position {
  x: number;
  y: number;
  z: number;
}

export interface CharacterState {
  position: Position;
  isMoving: boolean;
}

export interface DialogState {
  text: string;
  visible: boolean;
}

export interface GameState {
  character: CharacterState;
  dialog: DialogState;
  isLoading: boolean;
  isMuted: boolean;
  volume: number;
  playSoundCallback: (() => void) | null;

  // Actions
  setCharacterPosition: (pos: { x: number; z: number }) => void;
  setCharacterMoving: (moving: boolean) => void;
  setDialogText: (text: string) => void;
  setDialogVisible: (visible: boolean) => void;
  setIsLoading: (loading: boolean) => void;
  toggleMute: () => void;
  registerSoundCallback: (cb: () => void) => void;
  playSound: () => void;
}
```

**功能说明**：
- 定义Position接口，表示位置坐标
- 定义CharacterState接口，表示角色状态
- 定义DialogState接口，表示对话状态
- 定义GameState接口，包含所有游戏状态和操作方法

## 4. 技术实现细节

### 4.1 3D渲染实现

- 使用React Three Fiber作为React和Three.js的桥梁
- 使用React Three Drei提供的Billboard组件实现角色始终面向相机的效果
- 使用useLoader加载纹理资源，确保纹理加载完成后再渲染
- 配置适当的光照，使角色和场景可见

### 4.2 状态管理实现

- 使用Zustand作为轻量级状态管理库
- 采用切片式状态管理，将不同类型的状态分开管理
- 提供清晰的状态更新方法，确保状态更新的一致性
- 支持状态的持久化和响应式更新

### 4.3 纹理加载实现

- 将纹理文件放置在public/textures目录下
- 使用相对路径访问纹理资源
- 实现纹理加载的错误处理，确保组件不会因纹理加载失败而崩溃
- 支持纹理的透明效果，增强视觉表现力
