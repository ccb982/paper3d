# 角色实体函数文档

## 1. 实体系统架构

### 1.1 核心类层次结构

```
Entity (基类)
└── CharacterEntity (角色基类)
    ├── FriendlyEntity (友方实体)
    └── EnemyEntity (敌方实体)
```

### 1.2 系统组成

- **EntityManager**: 管理所有实体的生命周期
- **Entity**: 所有动态对象的基类
- **CharacterEntity**: 所有角色的基类
- **FriendlyEntity**: 友方角色（玩家控制或友方AI）
- **EnemyEntity**: 敌方角色（敌方AI）
- **PlayerCharacterManager**: 专门管理当前被操控角色的容器

### 3.3 可射击属性

所有角色实体都具有 `isShootable: true` 属性，这使得：
- 敌人会被射击系统识别为可射击目标
- 锁定式射击系统会将敌人作为锁定目标
- 自由式射击系统会对敌人进行射线检测
- 子弹会与敌人发生碰撞

### 4.3 角色直面摄像机功能

FriendlyEntity 实现了让玩家控制的角色始终背对摄像机（直面玩家）的功能：

```typescript
private faceToCamera(delta: number): void {
  if (!this.cameraRef || !this.mesh) return;

  const cameraPos = this.cameraRef.position.clone();
  const characterPos = this.mesh.position.clone();

  // 计算从角色指向相机的水平方向
  const toCamera = new THREE.Vector3().subVectors(cameraPos, characterPos);
  toCamera.y = 0;
  toCamera.normalize();

  // 角色背面朝向相机 → 角色的正面方向 = 相机方向的相反数
  const targetDirection = toCamera.clone().negate();

  // 计算目标旋转（只绕Y轴）
  const targetQuat = new THREE.Quaternion().setFromUnitVectors(
    new THREE.Vector3(0, 0, 1),  // 角色默认正面是 +Z
    targetDirection
  );

  // 平滑旋转
  const rotationSpeed = 8.0;
  this.mesh.quaternion.rotateTowards(targetQuat, rotationSpeed * delta);
  this.mesh.quaternion.normalize();
}
```

## 6. 玩家角色管理器 (PlayerCharacterManager)

### 6.1 功能说明

PlayerCharacterManager 是一个专门管理"当前被操控角色"的容器，采用单例模式实现。它提供了快速访问当前玩家角色的接口，便于其他系统（如输入系统、射击系统）获取和操作当前角色。

### 6.2 主要方法

| 方法 | 参数 | 返回值 | 描述 |
|------|------|--------|------|
| `getInstance()` | 无 | PlayerCharacterManager | 获取单例实例 |
| `setCurrentCharacter(character)` | character: CharacterEntity \\| null | 无 | 设置当前被玩家操控的角色 |
| `getCurrentCharacter()` | 无 | CharacterEntity \\| null | 获取当前被玩家操控的角色 |
| `updateCurrentCharacterPosition(x, y, z)` | x: number, y: number, z: number | 无 | 更新当前角色的位置 |
| `getCurrentCharacterId()` | 无 | string \\| null | 获取当前角色ID |
| `isCurrentCharacter(characterId)` | characterId: string | boolean | 判断指定ID是否为当前角色 |

### 6.3 使用示例

```typescript
import { playerCharacterManager } from '../systems/character/PlayerCharacterManager';

// 设置当前操控角色
playerCharacterManager.setCurrentCharacter(playerEntity);

// 获取当前操控角色
const currentChar = playerCharacterManager.getCurrentCharacter();

// 更新当前角色位置
playerCharacterManager.updateCurrentCharacterPosition(x, y, z);

// 判断是否为当前角色
if (playerCharacterManager.isCurrentCharacter('player')) {
  // ...
}
```

### 6.4 在 GameWorld 中的集成

```typescript
// GameWorld.tsx - 实体创建时注册
useEffect(() => {
  // 创建玩家实体
  const playerEntity = new FriendlyEntity('player', '/textures/character.png', position);
  entityManager.addEntity(playerEntity);
  
  // 注册到玩家角色管理器
  playerCharacterManager.setCurrentCharacter(playerEntity);
}, []);

// 每帧更新时传递相机引用
useFrame(({ camera }, delta) => {
  const currentChar = playerCharacterManager.getCurrentCharacter();
  if (currentChar && 'setCamera' in currentChar) {
    (currentChar as FriendlyEntity).setCamera(camera);
  }
  
  EntityManager.getInstance().update(delta);
});
```

## 8. 使用示例

### 8.1 创建友方实体（玩家）

```typescript
import { FriendlyEntity } from './src/entities/FriendlyEntity';
import { playerCharacterManager } from './src/systems/character/PlayerCharacterManager';
import * as THREE from 'three';

// 创建玩家实体
const player = new FriendlyEntity(
  'player',
  '/textures/character.png',
  new THREE.Vector3(0, 0, 0)
);

// 添加到实体管理器
EntityManager.getInstance().addEntity(player);

// 注册到玩家角色管理器
playerCharacterManager.setCurrentCharacter(player);
```

### 8.2 创建敌方实体

```typescript
import { EnemyEntity } from './src/entities/EnemyEntity';
import * as THREE from 'three';

// 创建敌方实体
const enemy = new EnemyEntity(
  'enemy1',
  '/textures/character.png',
  new THREE.Vector3(5, 0, 5)
);

// 添加到实体管理器
EntityManager.getInstance().addEntity(enemy);
```

### 8.3 实体更新

```typescript
// 在游戏主循环中调用
function animate() {
  const delta = clock.getDelta();

  // 更新所有实体
  EntityManager.getInstance().update(delta);

  renderer.render(scene, camera);
  requestAnimationFrame(animate);
}
```

## 9. 核心功能说明

### 9.1 纹理加载

- 支持异步加载纹理
- 加载失败时会输出错误信息
- 可以动态更换纹理

### 9.2 移动系统

- 支持玩家控制和AI控制
- 实现了基本的移动逻辑
- 支持重力效果

### 9.3 攻击系统

- 实现了攻击冷却机制
- 子类可以重写攻击逻辑

### 9.4 碰撞检测

- 实体具有碰撞半径
- 可以通过实体管理器获取实体进行碰撞检测

### 9.5 事件系统

- 角色死亡时会触发 'entity:death' 事件

### 9.6 点击检测

- 为角色添加了点击检测相关属性
- 可以通过射线检测实现角色点击交互

### 9.7 角色直面摄像机

- 玩家控制的角色始终背对摄像机（直面玩家）
- 使用平滑旋转实现自然的转向效果
- 旋转速度为 8.0 弧度/秒

### 9.8 玩家角色管理

- 通过 PlayerCharacterManager 统一管理当前操控角色
- 其他系统可以快速获取和操作当前角色
- 支持角色切换功能

## 10. 代码优化建议

1. **性能优化**：
   - 对于大量实体的场景，考虑使用空间分区算法（如四叉树）优化碰撞检测
   - 纹理加载可以实现预加载和缓存机制

2. **功能扩展**：
   - 添加角色状态系统（如中毒、眩晕等）
   - 实现角色技能系统
   - 添加角色动画系统

3. **代码结构**：
   - 考虑将AI逻辑抽象为独立的组件
   - 实现状态模式管理角色行为

4. **安全性**：
   - 添加实体属性验证
   - 实现边界检查，防止实体移出场景

## 11. 总结

角色实体系统是游戏的核心组成部分，通过继承和多态实现了不同类型角色的统一管理。系统设计清晰，功能完善，为游戏提供了良好的扩展性和可维护性。

通过实体管理器，可以方便地创建、更新和销毁角色实体，而角色基类提供了统一的接口和基础功能，使得不同类型的角色可以共享相同的核心逻辑，同时通过子类重写实现各自的特殊行为。

PlayerCharacterManager 的引入进一步简化了当前操控角色的管理，使得其他系统可以快速访问和操作当前玩家角色，无需遍历或状态查询。

这种设计不仅简化了代码结构，也提高了代码的可重用性和可维护性，为游戏的后续开发奠定了坚实的基础。
