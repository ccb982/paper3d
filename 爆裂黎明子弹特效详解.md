# 爆裂黎明子弹特效详解

## 概述

本文档详细说明了爆裂黎明子弹的子弹头形状、尾焰形状、绑定方式以及绑定后的视觉效果。

## 1. 子弹头形状

### 1.1 几何体结构

子弹头使用**水滴状几何体**，通过Three.js的`CylinderGeometry`创建：

```typescript
const geometry = new THREE.CylinderGeometry(0.1, 0.3, 0.6, 12);
```

**参数说明**：
- `0.1`：顶部半径（较小）
- `0.3`：底部半径（较大）
- `0.6`：高度
- `12`：径向分段数（圆柱体的边数）

**形状特点**：
- 顶部小、底部大，形成水滴形状
- 高度为0.6单位
- 整体呈圆锥状，顶部尖锐，底部宽阔

### 1.2 材质属性

```typescript
const material = new THREE.MeshStandardMaterial({
  color: 0xdd5b42,        // 颜色：#dd5b42（橙红色）
  emissive: 0xdd5b42,     // 自发光颜色
  emissiveIntensity: 1.0  // 自发光强度
});
```

**材质特点**：
- 基础颜色为橙红色（#dd5b42）
- 带有自发光效果，使子弹在暗处也清晰可见
- 使用标准材质，支持光照计算

### 1.3 朝向调整

子弹在飞行时会自动调整朝向，使尾部朝向飞行方向：

```typescript
// 计算子弹应该朝向的方向（反方向，让尾部朝向飞行方向）
const direction = this.velocity.clone().normalize().negate();
const up = new THREE.Vector3(0, 1, 0);
const quaternion = new THREE.Quaternion().setFromUnitVectors(up, direction);
this.mesh.quaternion.copy(quaternion);
```

## 2. 子弹尾焰形状

### 2.1 双层尾气几何体

子弹头部绑定了一个**双层尾气几何体**，每层使用不同的摆动算法：

#### 2.1.1 几何体形状

尾气使用**三角形拼接几何体**，由以下点集构成：

```typescript
const rawPoints: [number, number][] = [
  [761.93, 438.69],  // 中心点（头部）
  [754.23, 469.47],  // 周围点
  [743.97, 510.52],
  // ... 更多点
  [774.76, 405.34]   // 最后一个点
];
```

**形状特点**：
- 中心点：[761.93, 438.69]（头部位置）
- 周围点：围绕中心点形成不规则形状
- 归一化后：所有点映射到[0, 1]范围

#### 2.1.2 几何体构造

```typescript
for (let layer = 0; layer < 2; layer++) {
  const zOffset = layer * 0.1;
  
  for (let i = 1; i < normalizedPoints.length - 1; i++) {
    const center = normalizedPoints[0];
    const current = normalizedPoints[i];
    const next = normalizedPoints[i + 1];
    
    // 创建三角形
    vertices.push(
      center[0], center[1], zOffset,
      current[0], current[1], zOffset,
      next[0], next[1], zOffset
    );
  }
}
```

**构造方式**：
- **双层结构**：两层几何体，Z轴偏移0.1单位
- **三角形拼接**：从中心点到每两个相邻点创建三角形
- **平面结构**：厚度为0，无立体效果

#### 2.1.3 纹理颜色

尾气纹理包含**多层颜色渐变**和**噪点效果**：

**渐变颜色**：
```typescript
gradient.addColorStop(0, 'rgba(1, 1, 3, 1)');        // 头部：接近黑色
gradient.addColorStop(0.5, 'rgba(20, 25, 40, 1)');   // 上半部分结束：蓝色
gradient.addColorStop(0.55, 'rgba(60, 25, 40, 1)');  // 过渡开始
gradient.addColorStop(0.6, 'rgba(127, 24, 39, 1)');  // 过渡中
gradient.addColorStop(0.65, 'rgba(163, 27, 43, 1)'); // 过渡继续
gradient.addColorStop(0.7, 'rgba(200, 30, 48, 1)');  // 下半部分开始：#a31827
gradient.addColorStop(0.95, 'rgba(200, 30, 48, 1)'); // 下半部分
gradient.addColorStop(1, 'rgba(254, 122, 145, 1)');  // 最底部：#fe7a91
```

**颜色区域**：
- **上半部分（0-0.5）**：深蓝到蓝色渐变
- **过渡区域（0.5-0.7）**：蓝色到#a31827渐变
- **下半部分（0.7-0.95）**：#a31827（红色）
- **最底部（0.95-1.0）**：#fe7a91（粉红色）

**噪点效果**：
- **上半部分**：蓝色到紫色范围（R:10-40, G:15-25, B:25-60）
- **圆形区域**：#fe3362附近（R:230-255, G:60-110, B:90-140）
  - 位置：中心(x=50%, y=90%)
  - 半径：纹理尺寸的25%
- **最底部区域**：#fe7a91附近（R:220-255, G:100-160, B:130-190）
  - 范围：0.95-1.0
- **下半部分其他区域**：#a31827附近（R:150-250, G:20-50, B:40-70）

#### 2.1.4 双层不同摆动算法

尾气通过着色器实现**头部固定、尾部摆动**的效果，两层使用不同的摆动算法：

**第一层（layer=0）：快速小幅摆动**
```glsl
float tailFactor = position.y;
float wobbleAmplitude = tailFactor * tailFactor * uWobbleIntensity * 2.0;

// 第一层：快速小幅摆动
float wobbleX = sin(uTime * uWobbleSpeed * 1.5 + position.y * 12.0) * wobbleAmplitude * 0.8;
float wobbleZ = cos(uTime * uWobbleSpeed * 1.2 + position.y * 10.0) * wobbleAmplitude * 0.6;
```

**第二层（layer=1）：慢速大幅摆动**
```glsl
float tailFactor = position.y;
float wobbleAmplitude = tailFactor * tailFactor * uWobbleIntensity * 2.0;

// 第二层：慢速大幅摆动
float wobbleX = sin(uTime * uWobbleSpeed * 0.8 + position.y * 8.0) * wobbleAmplitude * 1.2;
float wobbleZ = cos(uTime * uWobbleSpeed * 0.6 + position.y * 6.0) * wobbleAmplitude * 1.0;
```

**摆动特点对比**：

| 参数 | 第一层 | 第二层 |
|-----|-------|-------|
| X轴速度 | 1.5倍 | 0.8倍 |
| Z轴速度 | 1.2倍 | 0.6倍 |
| X轴幅度 | 0.8倍 | 1.2倍 |
| Z轴幅度 | 0.6倍 | 1.0倍 |
| Y轴频率 | 12.0 | 8.0 |
| Y轴频率 | 10.0 | 6.0 |

**整体效果**：
- **头部固定**：y=0处摆动幅度为0
- **尾部摆动**：y=1处摆动幅度最大
- **双层叠加**：两层摆动叠加，形成复杂的动态效果
- **立体摆动**：X轴和Z轴同时摆动，实现立体效果

## 3. 绑定方式

### 3.1 尾气绑定

尾气通过**位置和朝向**绑定到子弹头部：

```typescript
// 创建尾气
const flightDirection = this.velocity.clone().normalize();
const trailOffset = flightDirection.clone().multiplyScalar(0.5);
this.trailMesh.position.copy(this.position).add(trailOffset);

// 设置朝向：头部朝向飞行方向（下面是头）
this.trailMesh.lookAt(this.position.clone().add(flightDirection));
this.trailMesh.rotateX(Math.PI); // 旋转180度，让头部朝前
```

**更新逻辑**：
```typescript
if (this.trailMesh && this.trailMaterial) {
  const flightDirection = this.velocity.clone().normalize();
  
  const trailOffset = flightDirection.clone().multiplyScalar(0.5);
  this.trailMesh.position.copy(this.position).add(trailOffset);
  
  this.trailMesh.lookAt(this.position.clone().add(flightDirection));
  
  this.trailMaterial.uniforms.uTime.value = this.trailTime;
}
```

**绑定特点**：
- **位置绑定**：尾气位于子弹前方0.5单位处
- **朝向绑定**：尾气头部朝向飞行方向
- **旋转调整**：旋转180度，使尾气头部（y=0）朝前
- **时间更新**：每帧更新着色器时间，实现摆动动画

## 6. 性能优化

### 6.1 纹理优化

- 使用CanvasTexture生成纹理，避免加载外部图片
- 纹理尺寸：512x512，平衡质量和性能
- 纹理复用：通过TextureManager管理，避免重复创建

### 6.2 几何体优化

- 使用BufferGeometry，减少内存占用
- 三角形拼接，减少顶点数量
- 双层结构，增加视觉层次

### 6.3 渲染优化

- 使用ShaderMaterial，GPU加速渲染
- 着色器中实现摆动动画，减少CPU计算
- 批量更新，减少渲染调用

4. **修改摆动效果**：调整着色器中的摆动参数

## 8. 双层摆动算法详解

### 8.1 算法原理

双层摆动算法通过叠加两层不同频率和幅度的正弦波，创造出复杂的动态效果。

**第一层算法**：
```glsl
// 快速小幅摆动
float wobbleX = sin(uTime * uWobbleSpeed * 1.5 + position.y * 12.0) * wobbleAmplitude * 0.8;
float wobbleZ = cos(uTime * uWobbleSpeed * 1.2 + position.y * 10.0) * wobbleAmplitude * 0.6;
```

**第二层算法**：
```glsl
// 慢速大幅摆动
float wobbleX = sin(uTime * uWobbleSpeed * 0.8 + position.y * 8.0) * wobbleAmplitude * 1.2;
float wobbleZ = cos(uTime * uWobbleSpeed * 0.6 + position.y * 6.0) * wobbleAmplitude * 1.0;
```

### 8.2 算法特点

**时间维度**：
- 第一层：快速变化（1.5倍和1.2倍速度）
- 第二层：慢速变化（0.8倍和0.6倍速度）

**空间维度**：
- 第一层：高频变化（12.0和10.0频率）
- 第二层：低频变化（8.0和6.0频率）

**幅度维度**：
- 第一层：小幅摆动（0.8倍和0.6倍幅度）
- 第二层：大幅摆动（1.2倍和1.0倍幅度）

### 8.3 叠加效果

两层摆动叠加后，尾气会产生：
- **快速抖动**：来自第一层的高频小幅摆动
- **慢速摇摆**：来自第二层的低频大幅摆动
- **复杂波形**：两层正弦波叠加形成非周期性运动
- **立体效果**：X轴和Z轴同时摆动

## 9. 总结

爆裂黎明子弹特效由以下部分组成：

1. **子弹头**：水滴状几何体，橙红色，自发光
2. **尾气**：双层几何体，带有渐变、噪点和双层不同摆动算法

绑定方式：
- 尾气：通过位置和朝向绑定到子弹头部

视觉效果：
- 多层次颜色分布
- 丰富的动态效果
- 立体的双层摆动动画

性能优化：
- 纹理复用
- 几何体优化
- GPU加速渲染

扩展性：
- 可自定义参数
- 易于添加新效果
- 模块化设计
