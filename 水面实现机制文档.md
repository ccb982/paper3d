# 水面实现机制文档

## 1. 概述

本文档详细介绍了游戏中水面实体的实现机制，包括核心架构、技术原理、代码结构和使用方法。

## 2. 核心架构

水面实体 `WaterEntity` 是一个基于 Three.js 的动态水面实现，采用了波动方程来模拟水面效果。

### 2.1 类结构

```typescript
export class WaterEntity extends StaticEntity {
  private width: number;           // 水面宽度
  private height: number;          // 水面高度
  private resolution: number;      // 高度场分辨率
  private rtA: THREE.WebGLRenderTarget;  // 高度场纹理A
  private rtB: THREE.WebGLRenderTarget;  // 高度场纹理B
  private computeMaterial: THREE.ShaderMaterial;  // 波动方程计算材质
  private fullscreenPlane: THREE.Mesh;  // 全屏平面用于渲染到纹理

  private waterMaterial: THREE.ShaderMaterial;  // 水面渲染材质
  private gridSegments: number;    // 水面网格分段数
}
```

## 3. 技术原理

### 3.1 双缓冲纹理技术

当前实现使用了**双缓冲纹理**（Ping-Pong Buffer）技术来模拟水面波动：

- **rtA** 和 **rtB**：两个 `WebGLRenderTarget` 用于存储高度场数据
- **分辨率**：默认 256x256，可通过构造函数参数调整
- **数据格式**：使用 `FloatType` 和 `RedFormat`，每个像素存储一个高度值

### 3.2 波动方程实现

核心使用**波动方程**来计算水面的运动：

```typescript
// 波动方程计算材质的片元着色器
fragmentShader: `
  uniform sampler2D uHeightNow;
  uniform sampler2D uHeightPrev;
  uniform float uLambda2;
  uniform vec2 uTexelSize;
  varying vec2 vUv;

  void main() {
    vec2 uv = vUv;
    float h = texture2D(uHeightNow, uv).r;
    float hL = texture2D(uHeightNow, uv + vec2(-uTexelSize.x, 0)).r;
    float hR = texture2D(uHeightNow, uv + vec2(uTexelSize.x, 0)).r;
    float hT = texture2D(uHeightNow, uv + vec2(0, uTexelSize.y)).r;
    float hB = texture2D(uHeightNow, uv + vec2(0, -uTexelSize.y)).r;
    float hPrev = texture2D(uHeightPrev, uv).r;

    float laplacian = (hL + hR + hT + hB - 4.0 * h);
    float hNext = 2.0 * h - hPrev + uLambda2 * laplacian;

    gl_FragColor = vec4(hNext, 0.0, 0.0, 1.0);
  }
`
```

**波动方程解释**：
- `h`：当前点的高度
- `hL`, `hR`, `hT`, `hB`：相邻四个点的高度
- `hPrev`：上一帧的高度
- `laplacian`：拉普拉斯算子，计算高度场的二阶导数
- `hNext`：下一帧的高度，通过波动方程计算得出

## 5. 关键方法

### 5.1 高度场初始化

```typescript
private initHeightTexture(): THREE.DataTexture {
  const data = new Float32Array(this.resolution * this.resolution);
  for (let i = 0; i < this.resolution; i++) {
    for (let j = 0; j < this.resolution; j++) {
      const x = (i / this.resolution) * 2 - 1;
      const y = (j / this.resolution) * 2 - 1;
      const r = Math.sqrt(x * x + y * y);
      // 在中心创建一个初始鼓包
      const value = r < 0.2 ? Math.cos(r * Math.PI * 5) * 0.1 : 0;
      data[i + j * this.resolution] = value;
    }
  }
  const tex = new THREE.DataTexture(data, this.resolution, this.resolution, THREE.RedFormat, THREE.FloatType);
  tex.needsUpdate = true;
  return tex;
}
```

### 7.3 替代方案：使用 ShaderMaterial

另一种更高效的方案是使用 `ShaderMaterial` 直接在着色器中实现波动效果，而不是修改顶点数据：

```typescript
// 创建水面材质 - 使用ShaderMaterial实现波动效果
this.waterMaterial = new THREE.ShaderMaterial({
  uniforms: {
    uTime: { value: 0 },
    uWidth: { value: this.width },
    uHeightMap: { value: this.rtA.texture }
  },
  vertexShader: `
    uniform float uTime;
    uniform float uWidth;
    uniform sampler2D uHeightMap;
    varying vec2 vUv;
    varying vec3 vNormal;

    void main() {
      vUv = uv;
      
      // 从高度纹理获取高度
      float height = texture2D(uHeightMap, vUv).r;
      vec3 newPosition = vec3(position.x, height, position.z);
      
      // 计算法线（简化版）
      vNormal = vec3(0, 1, 0);
      
      vec4 mvPosition = modelViewMatrix * vec4(newPosition, 1.0);
      gl_Position = projectionMatrix * mvPosition;
    }
  `,
  fragmentShader: `
    uniform float uTime;
    varying vec2 vUv;
    varying vec3 vNormal;

    void main() {
      // 基础颜色 - 亮蓝色
      vec3 baseColor = vec3(0.3, 0.6, 1.0);
      
      // 模拟水面反射和高光
      vec3 lightDir = normalize(vec3(1.0, 1.0, 1.0));
      float diff = max(dot(vNormal, lightDir), 0.0);
      vec3 reflectDir = reflect(-lightDir, vNormal);
      vec3 viewDir = normalize(vec3(0.0, 1.0, 1.0));
      float spec = pow(max(dot(reflectDir, viewDir), 0.0), 32.0);
      
      // 添加水面波动的微小颜色变化
      float waveEffect = sin(vUv.x * 10.0 + uTime) * 0.1;
      vec3 finalColor = baseColor * (0.8 + diff * 0.2) + spec * 0.5 + waveEffect;
      
      gl_FragColor = vec4(finalColor, 1.0);
    }
  `,
  transparent: true,
  side: THREE.DoubleSide
});
```

## 8. 性能优化

### 8.1 纹理分辨率

- **默认分辨率**：256x256
- **优化建议**：
  - 对于小型水面（< 50x50），可以使用 128x128 分辨率
  - 对于大型水面（> 200x200），可以使用 512x512 分辨率
  - 分辨率越高，波动效果越细腻，但性能消耗也越大

### 8.2 网格分段数

- **默认分段数**：128
- **优化建议**：
  - 分段数应与水面尺寸匹配，一般保持每单位 1-2 个分段
  - 例如，20x20 的水面使用 32-64 个分段即可

### 8.3 渲染优化

- **使用帧缓冲对象**：减少纹理切换的开销
- **使用实例化渲染**：如果场景中有多个水面
- **LOD 技术**：根据相机距离调整水面的细节
