目前已实现了一个静态水面，但希望增加随机波动效果，让水面看起来更生动（如微风拂过、随机涟漪等）。根据你提供的《水面实现机制文档》，当前水面波动计算部分（updateWave）被暂时禁用，导致水面保持静态。要实现随机波动，我们需要：

恢复波动模拟（基于双缓冲纹理+波动方程），确保水面能动态变化；

注入随机扰动，在随机位置、随机时间向高度场添加额外的能量，产生自然的随机波动；

将高度场实时反映到渲染，让水面几何体或材质根据高度纹理变形。

下面我会提供一套完整的、可直接集成的实现方案，基于你已有的 WaterEntity 类进行扩展，采用 ShaderMaterial 顶点纹理读取 的方式，避免 CPU 修改几何体带来的性能问题和消失风险。

一、整体思路
每帧更新高度场纹理（双缓冲波动方程）—— 你已有 rtA/rtB 和 computeMaterial，只需启用并正确执行。

随机注入扰动：在每帧或间隔几帧，向高度场添加随机位置的“鼓包”或“尖峰”，模拟随机外力（如雨滴、鱼跃、风）。

渲染时采样高度纹理：水面网格不再使用固定几何体，而是通过 ShaderMaterial 在顶点着色器中读取高度纹理，实时改变顶点 Y 坐标，并计算简易法线用于光照。

这样做的好处：

无需每帧从 GPU 读回高度数据；

波动计算和渲染完全在 GPU 进行，性能极佳；

随机扰动可以轻松注入（通过叠加一个额外的扰动纹理或直接修改当前高度场纹理的某几个像素）。

二、具体实现步骤
步骤 1：恢复波动方程计算循环
取消 updateWave 方法的注释，并补齐渲染到纹理的逻辑。同时需要在每一帧调用 updateWave。

修改 WaterEntity 如下：

typescript
private updateWave(): void {
  const renderer = EntityManager.getInstance().getRenderer();
  if (!renderer) return;

  // 交换双缓冲纹理：当前帧成为上一帧，计算下一帧到 rtB
  const temp = this.rtA;
  this.rtA = this.rtB;
  this.rtB = temp;

  // 更新材质 uniforms
  this.computeMaterial.uniforms.uHeightNow.value = this.rtA.texture;
  this.computeMaterial.uniforms.uHeightPrev.value = this.rtB.texture;

  // 渲染到 rtB（下一帧的高度场）
  renderer.setRenderTarget(this.rtB);
  renderer.render(this.sceneRTT, this.cameraRTT);
  renderer.setRenderTarget(null);
}

public update(delta: number): void {
  super.update(delta);
  this.updateWave();                 // 每帧更新波动方程
  this.addRandomDisturbances();      // 随机注入扰动（下一步实现）
}
注意：你需要确保场景 sceneRTT 中包含那个全屏平面（fullscreenPlane），并且相机 cameraRTT 设置正确。这些在你的构造函数中已经创建，只需确认 sceneRTT.add(fullscreenPlane) 和 cameraRTT 为正交相机（left/right = -1/1，top/bottom = -1/1）。

步骤 2：实现随机扰动注入
由于我们无法直接在 GPU 纹理上高效随机写像素，最简单可靠的方法是在波动方程计算材质中增加一个“外力”项：每帧随机生成一组扰动位置和强度，通过 uniform 数组传递给片元着色器，在计算 hNext 时叠加。

然后在 updateWave 中，每帧随机生成 0~3 个扰动点，更新 uniform 数组，并传递给着色器。同时在片元着色器中判断当前像素是否靠近扰动点，若靠近则增加高度。

片段着色器关键修改（在原波动方程基础上增加扰动叠加）：

glsl
uniform sampler2D uHeightNow;
uniform sampler2D uHeightPrev;
uniform float uLambda2;
uniform float uDamping;
uniform vec2 uTexelSize;
uniform int uDisturbCount;
uniform vec2 uDisturbPositions[20];
uniform float uDisturbStrengths[20];
uniform float uTime;
varying vec2 vUv;

void main() {
  vec2 uv = vUv;
  float h = texture2D(uHeightNow, uv).r;
  float hL = texture2D(uHeightNow, uv + vec2(-uTexelSize.x, 0)).r;
  float hR = texture2D(uHeightNow, uv + vec2( uTexelSize.x, 0)).r;
  float hT = texture2D(uHeightNow, uv + vec2(0,  uTexelSize.y)).r;
  float hB = texture2D(uHeightNow, uv + vec2(0, -uTexelSize.y)).r;
  float hPrev = texture2D(uHeightPrev, uv).r;

  float laplacian = (hL + hR + hT + hB - 4.0 * h);
  float hNext = 2.0 * h - hPrev + uLambda2 * laplacian;
  hNext *= uDamping;

  // 添加随机扰动
  float disturbance = 0.0;
  for (int i = 0; i < 20; i++) {
    if (i >= uDisturbCount) break;
    vec2 delta = uv - uDisturbPositions[i];
    float dist2 = dot(delta, delta);
    float radius = 0.02;          // 扰动影响半径（相当于物理尺寸 2% 的水面区域）
    if (dist2 < radius) {
      float strength = uDisturbStrengths[i];
      // 高斯型扰动，中心最强
      disturbance += strength * (1.0 - smoothstep(0.0, radius, dist2));
    }
  }
  hNext += disturbance;

  gl_FragColor = vec4(hNext, 0.0, 0.0, 1.0);
}
在 WaterEntity 中添加方法，每帧更新随机扰动参数：

typescript
private randomDisturbances: { pos: THREE.Vector2, strength: number }[] = [];

private addRandomDisturbances(): void {
  const now = performance.now() / 1000;
  // 每秒随机添加 1~4 次扰动
  const chance = 0.02; // 每帧概率
  if (Math.random() < chance && this.randomDisturbances.length < 10) {
    const randomX = (Math.random() - 0.5) * 0.8; // UV坐标范围[0,1]，避开边缘
    const randomY = (Math.random() - 0.5) * 0.8;
    const pos = new THREE.Vector2(0.5 + randomX, 0.5 + randomY);
    const strength = 0.05 + Math.random() * 0.1;
    this.randomDisturbances.push({ pos, strength });
  }
  // 衰减扰动强度，并移除过弱的
  for (let i = 0; i < this.randomDisturbances.length; i++) {
    this.randomDisturbances[i].strength *= 0.98;
    if (this.randomDisturbances[i].strength < 0.005) {
      this.randomDisturbances.splice(i,1);
      i--;
    }
  }
  // 传递给材质
  const count = Math.min(this.randomDisturbances.length, 20);
  const positions = new Array(20).fill(new THREE.Vector2(0,0));
  const strengths = new Array(20).fill(0);
  for (let i = 0; i < count; i++) {
    positions[i] = this.randomDisturbances[i].pos;
    strengths[i] = this.randomDisturbances[i].strength;
  }
  this.computeMaterial.uniforms.uDisturbCount.value = count;
  this.computeMaterial.uniforms.uDisturbPositions.value = positions;
  this.computeMaterial.uniforms.uDisturbStrengths.value = strengths;
  this.computeMaterial.uniforms.uTime.value = performance.now() / 1000;
}
这样，每帧都有一定概率在随机 UV 位置产生一个逐渐衰减的扰动鼓包，波动方程会自动将这些扰动传播开来，形成随机涟漪。

步骤 3：替换水面材质为 ShaderMaterial（支持高度纹理采样）
移除原来的 MeshPhongMaterial，改用 ShaderMaterial，在顶点着色器里读取高度纹理 (当前最新高度场 rtA.texture)，修改顶点 Y 坐标，并传递 UV 和法线。

在 createWaterEffect 中创建水面网格的部分：

typescript
// 创建水面几何体（平面，细分足够）
const geometry = new THREE.PlaneGeometry(this.width, this.height, this.gridSegments, this.gridSegments);
geometry.rotateX(-Math.PI / 2); // 转为水平面

// 水面对着色器材质
this.waterMaterial = new THREE.ShaderMaterial({
  uniforms: {
    uHeightMap: { value: this.rtA.texture },   // 高度纹理
    uTime: { value: 0 },
    uColorA: { value: new THREE.Color(0x2c7da0) },
    uColorB: { value: new THREE.Color(0x61a5c2) }
  },
  vertexShader: `
    uniform sampler2D uHeightMap;
    uniform float uTime;
    varying vec2 vUv;
    varying vec3 vViewPosition;

    void main() {
      vUv = uv;
      // 采样高度纹理，获取当前顶点高度偏移
      float height = texture2D(uHeightMap, uv).r;
      vec3 newPosition = position + vec3(0.0, height * 0.5, 0.0); // 调整强度系数
      
      vec4 mvPosition = modelViewMatrix * vec4(newPosition, 1.0);
      vViewPosition = -mvPosition.xyz;
      gl_PointSize = 1.0;
      gl_Position = projectionMatrix * mvPosition;
    }
  `,
  fragmentShader: `
    uniform vec3 uColorA;
    uniform vec3 uColorB;
    uniform float uTime;
    varying vec2 vUv;

    void main() {
      // 简单渐变 + 动态波纹光泽（模拟法线随uv变化）
      float wave = sin(vUv.x * 20.0 + uTime) * cos(vUv.y * 20.0 + uTime);
      vec3 color = mix(uColorA, uColorB, vUv.y + wave * 0.2);
      // 添加高光模拟（简单菲涅尔）
      float fresnel = pow(1.0 - abs(vUv.y - 0.5) * 2.0, 2.0);
      color += vec3(0.8, 0.9, 1.0) * fresnel * 0.5;
      gl_FragColor = vec4(color, 1.0);
    }
  `,
  transparent: true,
  side: THREE.DoubleSide
});

this.mesh = new THREE.Mesh(geometry, this.waterMaterial);
this.mesh.position.copy(this.position);
// 移除原来的临时 mesh，添加新水面
if (this.tempMesh) this.scene.remove(this.tempMesh);
this.scene.add(this.mesh);
注意：由于没有精确计算法线，光照是伪的。如需更真实的光照，可以在顶点着色器中采样高度纹理邻域计算法线，或者在片元着色器中使用差分方法。这里为了简洁先使用程序化波纹和渐变模拟。

每帧需要更新材质的 uniform uHeightMap 为当前高度纹理（双缓冲中即将用于渲染的那一个，通常是 rtA.texture，因为 rtA 是当前最新状态）。在 update 中同步：

typescript
public update(delta: number): void {
  super.update(delta);
  this.updateWave();
  this.addRandomDisturbances();
  // 更新水面材质的纹理引用和时间
  if (this.waterMaterial) {
    this.waterMaterial.uniforms.uHeightMap.value = this.rtA.texture;
    this.waterMaterial.uniforms.uTime.value = performance.now() / 1000;
  }
}
步骤 4：添加外部可控的随机扰动接口（可选）
你可以保留 addDisturbance 方法，让外部手动向指定位置添加扰动（例如点击水面触发涟漪）。

typescript
public addDisturbance(position: THREE.Vector2, strength: number = 0.1): void {
  // position 是相对于水面中心的世界坐标（x, z）
  // 转换为 UV 坐标
  const uv = new THREE.Vector2(
    (position.x + this.width / 2) / this.width,
    (position.z + this.height / 2) / this.height
  );
  if (uv.x >= 0 && uv.x <= 1 && uv.y >= 0 && uv.y <= 1) {
    this.randomDisturbances.push({ pos: uv, strength });
  }
}
三、参数调优建议
波动传播速度 uLambda2：0.3~0.6 效果较好，值越大波速越快，但可能不稳定。

阻尼 uDamping：0.99~0.998，让波动逐渐衰减，避免无限震荡。

扰动半径：上面示例中为 0.02（纹理 UV 空间），对应世界空间大约 width * 0.02。可根据水面尺寸适当调整。

随机扰动概率：每帧 0.02 约等于每秒产生 0.02×60≈1.2 个扰动，可根据想要的水面活跃度调节。

高度采样强度：顶点着色器中 height * 0.5，可改为 height * maxWaveHeight（例如 0.3~0.8），让波动幅度更明显。

四、最终效果
完成上述修改后，你的水面将具备：

自动波动传播：根据波动方程，高度场会自然演替，形成连续的涟漪。

随机扰动源：每隔几帧会在随机位置产生一个力量逐渐衰减的鼓包，产生随机、不重复的波动图案。

性能优化：所有计算都在 GPU 进行，没有 CPU 读回或顶点数组重建，即使分辨率 256x256、网格 128x128 也能流畅运行。
