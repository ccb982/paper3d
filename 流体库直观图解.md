# @bienehito/fluid-dynamics 直观图解

## 📌 关键概念：流体模拟就是解 6 个方程

```
┌─────────────────────────────────────────────────────────────┐
│  Navier-Stokes 方程（简化版）：                                │
│                                                             │
│  1.  ∂u/∂t + (u·∇)u = -∇p + ν∇²u + F   （动量方程）        │
│                                                             │
│  2.  ∇·u = 0                          （连续性方程，不可压） │
└─────────────────────────────────────────────────────────────┘
```

## 🎯 模拟流程（用 6 步解方程）

```
     ┌──────────────┐
     │  原始速度 u   │
     └────────┬─────┘
              │
              ▼
    ┌──────────────────┐
    │  1. curl(旋度)   │ ← 计算旋涡，为后续涡度力准备
    └────────┬─────────┘
              │
              ▼
    ┌──────────────────┐
    │ 2. vorticity力   │ ← 添加旋涡，让效果更有趣
    └────────┬─────────┘
              │
              ▼
    ┌──────────────────┐
    │  3. divergence   │ ← 计算"散度"：哪里有压缩/膨胀
    └────────┬─────────┘
              │
              ▼
    ┌──────────────────┐
    │  4. pressure     │ ← 解泊松方程（迭代多次！）
    └────────┬─────────┘
              │
              ▼
    ┌──────────────────┐
    │5. gradSubtract   │ ← 用压力修正速度，满足不可压
    └────────┬─────────┘
              │
              ▼
    ┌──────────────────┐
    │   6. advection   │ ← 传输速度和染料
    └────────┬─────────┘
              │
              ▼
     ┌──────────────┐
     │  更新后 u'   │
     └──────────────┘
```

## 📐 每个步骤做了什么（图解）

### 步骤 1 & 2：Curl + Vorticity（涡度力）

```
┌─────────────────────────────────────────────────────────┐
│ 想象有一条河：                                           │
│                                                           │
│        ████░░░░░░░░░░░░░░                                │
│        ████  >  >  >                                     │
│        ████░░░░░░░░░░░░░░                                │
│           ▲                                              │
│          障碍                                            │
│                                                           │
│ 水流绕过障碍 → 产生旋涡 → 用涡度力增强，让流体更"活泼"    │
└─────────────────────────────────────────────────────────┘
```

### 步骤 3：Divergence（散度）

```
散度 ∇·u = 单位体积的流出 - 流入

  ┌─────┐
  │     │
  │  •  │ →→
  │     │
  └─────┘
  ↑  ↑
  流入 流出

  ∇·u > 0 : 膨胀（源）
  ∇·u < 0 : 压缩（汇）
```

### 步骤 4：Pressure（压力求解）

```
这是最耗时的一步！解 Poisson 方程（用 Jacobi 迭代）

  初始：    P = 0

  迭代：    P = (L + R + T + B - div) / 4
                         ↓
                     周围邻居

  多次迭代后收敛到正确的压力！
```

### 步骤 5：Gradient Subtract（压力梯度）

```
从速度中减去压力梯度，保证不可压

  压力 P
   /|\
    │
    │
    │
    └───>  高→低

  压力梯度 ∇P = 指向高压区
```

### 步骤 6：Advection（平流/传输）

```
最简单也最直观！想象染料在流动：

  ┌────────────────────┐
  │        ⬤           │
  │        |           │
  │        v           │
  │                    │
  └────────────────────┘

  关键点：不是向前推，而是向后找！
  (为了数值稳定性)

  当前点 ←← 过去点

  coord = uv - dt * velocity * texelSize
```

## 🔧 数据结构：双缓冲纹理

```
为什么要双缓冲？

─── 读缓冲 ───      ─── 写缓冲 ───
   [ A ]     ──→     [ B ]
   (只读)            (只写)

然后：swap() ← 下次读 B，写 A

好处：消除数据依赖，GPU 并行计算更爽！
```

## 🎨 你调用的 API 做了什么

### `setDye(x, y, ...)` 调用流程：

```javascript
你：fd.setDye(x, y, 0, w, h, [0.2, 0.6, 0.9]);
 ↓
┌────────────────────────────────────┐
│  1. 转换坐标到 [0,1]                 │
│  2. 调用 splat shader 画高斯斑点     │
│  3. 写入 _dye.write FBO             │
│  4. swap()                          │
└────────────────────────────────────┘
 ↓
下次模拟时，这颗染料就会被平流 advection 带着走！
```

### `splat` 着色器：

```glsl
void main() {
    vec2 p = transform * (vUv - point);
    
    // ↓ 高斯函数 ↓
    float e = exp(-dot(p, p));
    
    vec3 base = texture2D(uSource, vUv).xyz;
    
    // 混合进去
    gl_FragColor = vec4(base*(1-e) + color*e, 1);
}
```

## ⚙️ TriangleFluidTexture 中的三角形是怎么工作的？

```javascript
for (let x = baseX; x <= actualTipX; x += 2) {
    const t = (x - baseX) / tallHeight;
    const halfHeight = size * 0.15 * (1 - t);
    
    // ← 关键：这里把速度设为 0！
    fluidDynamics.setVelocity(x, cy, 0, 3, halfHeight*2+1, 0, 0);
}
```

### 图解：

```
         尖端 ▲
              │
            ╱ ╲
           ╱   ╲
          ╱  x  ╲   ← 这里设为速度 0
         ╱       ╲
        ╱         ╲
       ╱           ╲
      ───────────────
            底部

水流过来 → 撞上设为 0 的区域 → 被"挡住" → 向两侧分流！
```

## 📚 推荐阅读顺序

2. 再懂 **divergence**（散度）
3. 再懂 **pressure**（压力求解，核心难点）
