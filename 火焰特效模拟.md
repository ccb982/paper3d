我们聚焦在粒子发射器和物理模拟器这两个核心模块，并考虑性能与复杂度控制。

一、粒子数据结构设计
为了性能，避免使用对象数组，改用并行数组（或单个 Float32Array 存储所有属性）。

typescript
// 最大粒子数
const MAX_PARTICLES = 3000;

// 属性数组（每个粒子一组）
const positions = new Float32Array(MAX_PARTICLES * 3); // x,y,z
const velocities = new Float32Array(MAX_PARTICLES * 3); // vx,vy,vz
const colors = new Float32Array(MAX_PARTICLES * 3);     // r,g,b
const sizes = new Float32Array(MAX_PARTICLES);
const lifetimes = new Float32Array(MAX_PARTICLES);     // 当前剩余寿命（秒）
const maxLifetimes = new Float32Array(MAX_PARTICLES);  // 初始最大寿命

let activeCount = 0; // 当前活跃粒子数
发射新粒子时，找到第一个 lifetimes[i] <= 0 的槽位复用，或使用环形队列。

二、粒子发射器
2.1 发射参数
typescript
interface EmitterConfig {
  rate: number;           // 每秒发射粒子数（射速）
  position: THREE.Vector3; // 发射源位置（世界坐标）
  shape: 'point' | 'sphere' | 'cone'; // 发射形状
  shapeRadius: number;    // 形状半径
  initialVelocity: THREE.Vector3; // 基础速度（向上为主）
  velocityRandomness: number;     // 速度随机范围（标量）
  initialColor: THREE.Color;      // 起始颜色
  finalColor: THREE.Color;        // 寿命结束颜色
  size: number;           // 粒子大小（世界单位）
  lifeMin: number;        // 最短寿命（秒）
  lifeMax: number;        // 最长寿命（秒）
}
2.2 发射逻辑
每帧根据 delta 时间计算应发射的粒子数（非整数时累积余数）。

typescript
let emitCount = 0;
this.emitAccumulator += delta * this.rate;
while (this.emitAccumulator >= 1.0 && activeCount < MAX_PARTICLES) {
  this.emitParticle();
  this.emitAccumulator -= 1.0;
  emitCount++;
}
emitParticle() 实现：

根据发射形状计算初始位置：

point: 直接使用 emitter.position

sphere: 在半径球体内随机 pos = emitter.position + randomInSphere(shapeRadius)

cone: 圆锥底部圆形区域随机，y 在底部附近。

初始速度 = 基础速度 + 随机偏移（三个轴乘以 velocityRandomness）。

颜色 = initialColor（随时间变化在物理模拟器中处理）。

寿命 = lifeMin + Math.random() * (lifeMax - lifeMin)。

将属性填入数组的空闲槽位，activeCount++。

三、物理模拟器
3.1 外力定义
火焰典型外力：

上升浮力：恒定加速度 (0, gravity, 0)，但重力通常向下，火焰应向上，所以 acc.y = +2.0 ~ +5.0。

湍流：随机加速度，可以用 Perlin 噪声或每个粒子独立的正弦/余弦波。

阻力：速度阻尼 velocity *= 0.99，让粒子减速并防止飞出太远。

风力（可选）：水平方向的整体加速度。

3.2 更新循环（每帧）
typescript
update(delta: number) {
  for (let i = 0; i < MAX_PARTICLES; i++) {
    if (lifetimes[i] <= 0) continue;

    // 1. 计算加速度
    let ax = 0, ay = 0, az = 0;
    // 上升浮力
    ay += this.upwardForce;
    // 湍流（每粒子独立随机相位）
    const phase = i * 0.01; // 伪随机，也可存储每个粒子的随机种子
    const time = performance.now() * 0.001;
    ax += Math.sin(time * 3.0 + phase) * this.turbulenceStrength;
    az += Math.cos(time * 2.5 + phase) * this.turbulenceStrength;
    // 可选：围绕中心轴的旋转湍流
    // 阻力
    velocities[i*3] *= this.damping;
    velocities[i*3+1] *= this.damping;
    velocities[i*3+2] *= this.damping;

    // 2. 更新速度
    velocities[i*3] += ax * delta;
    velocities[i*3+1] += ay * delta;
    velocities[i*3+2] += az * delta;

    // 3. 更新位置
    positions[i*3] += velocities[i*3] * delta;
    positions[i*3+1] += velocities[i*3+1] * delta;
    positions[i*3+2] += velocities[i*3+2] * delta;

    // 4. 更新寿命和颜色
    lifetimes[i] -= delta;
    const lifeFactor = lifetimes[i] / maxLifetimes[i]; // 1 -> 0
    // 颜色插值：从初始色到最终色
    const r = this.initialColor.r + (this.finalColor.r - this.initialColor.r) * (1 - lifeFactor);
    const g = this.initialColor.g + (this.finalColor.g - this.initialColor.g) * (1 - lifeFactor);
    const b = this.initialColor.b + (this.finalColor.b - this.initialColor.b) * (1 - lifeFactor);
    colors[i*3] = r;
    colors[i*3+1] = g;
    colors[i*3+2] = b;

    // 大小也可随寿命缩放
    sizes[i] = this.particleSize * (0.5 + lifeFactor * 0.5);
  }
}
3.3 控制复杂度：限制射速
你已经提到“如果特效过于复杂，可以考虑限制射速”。这确实是最直接有效的优化手段。

降低 rate（例如从 200 降到 80 粒子/秒），同时增加粒子寿命或增大粒子尺寸来补偿视觉密度。

配合粒子数量上限 MAX_PARTICLES，当活跃粒子数达到上限时，暂停发射直到有粒子消亡。

动态调整：在低端设备上，检测帧率，动态降低 rate。

四、与 Three.js 渲染集成
每帧更新完粒子属性后，更新 Points 对象的 buffer attribute 并重新上传：

typescript
const geometry = this.points.geometry;
geometry.attributes.position.needsUpdate = true;
geometry.attributes.color.needsUpdate = true;
geometry.attributes.size.needsUpdate = true; // 如果 PointsMaterial 支持 size 属性
PointsMaterial 默认不支持每个粒子不同大小，需要自定义 ShaderMaterial 或使用 THREE.Points + 纹理来实现。推荐使用 PointsMaterial + 纹理（圆形渐变纹理），所有粒子大小统一。若要每个粒子不同大小，需要写自定义着色器，在顶点着色器中读取 attribute size。

五、性能评估
3000 粒子，每帧执行约 3000 * (几十次浮点运算) ≈ 10 万次运算，现代设备轻松应对。

瓶颈可能在于 CPU↔GPU 数据传输（更新 buffer attribute）。但只要粒子数量 < 10000，基本无压力。

限制射速后，实际活跃粒子数会更低，性能更好。
