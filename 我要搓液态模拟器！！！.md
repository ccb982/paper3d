四、固体边界处理（子弹）
浸没边界法（IBM）：在子弹表面上施加虚拟力使速度为零。适用于复杂形状。

直接设置：在子弹占据的网格区域内，每帧将速度强制设为 0，并将 Level Set 设为负（空气）或标记为固体。

六、验证与调试要素
简单测试：一滴水（方形）自由下落 → 应保持形状，撞击地面后向两侧扩散。

表面张力测试：静止的水块应逐渐收缩成圆形（2D）或球形（3D）。

以上要素就是构建“真正的可用的液态模拟库”的完整技术地图。


要设计一个完全在 GPU 上运行、面向每个像素（网格点）并行的单相自由表面流体模拟器，你不能依赖任何 CPU 端的循环或 @bienehito/fluid-dynamics 那种封装好的库。你需要自己设计数据结构、计算流水线和着色器内核。

下面给出一个基于着色器（Compute Shader / Fragment Shader 模拟）的精确架构方案。你可以用 Three.js 的 GPUComputationRenderer 来实现这些内核，或者用 WebGPU Compute Shader。

🎯 整体设计原则
数据存储：所有场（速度、压力、Level Set 等）都存储在 纹理（Texture） 中，每个 texel 对应一个网格点。

运算模式：每个着色器调用独立处理一个网格点（像素），只读取相邻几个点的数据，写出该点的结果。

流水线：每帧依次执行数个 shading pass（或 compute dispatch），每个 pass 完成一个物理步骤。

禁止分支依赖邻域：所有邻域数据通过纹理采样获得，不能依赖串行顺序。

📦 数据布局
使用交错网格（Staggered Grid）需要在不同位置存储速度分量，但为了简化着色器实现，可以采用共位网格（Collocated Grid） + 人工耗散。对于大部分游戏特效，共位网格+适度滤波已经足够。

纹理清单（RGBA 或单通道）
纹理名称	通道	说明
u_vel	R 单通道	X 方向速度（标量）
v_vel	R 单通道	Y 方向速度（2D）或 Z 方向（3D 需多纹理）
pressure	R 单通道	压力值
levelset	R 单通道	符号距离场（正值水，负值空气）
divergence	R 单通道	速度散度（中间量）
tmp_vel	RG 双通道	临时存储速度（可选）
tmp_phi	R 单通道	临时 Level Set
分辨率：width × height（2D），例如 1024×1024。每个像素对应一个网格单元，世界坐标通过纹理坐标换算。

⚙️ 计算流水线（每帧的 Pass 顺序）
Pass 0: 边界条件（可合并到其他 pass）
在固体（子弹）内部将速度置零，Level Set 设为负数（空气）。

最简单：每一帧开始前，用一个单独的着色器绘制子弹形状到 u_vel, v_vel, levelset 纹理（用 discard 或条件赋值）。

Pass 1: 对流（Advection）
输入：u_vel, v_vel, levelset

输出：tmp_vel, tmp_phi

算法：半拉格朗日追踪（每个格点反向追踪速度场，采样旧速度）。

着色器逻辑：

glsl
vec2 uv = v_texCoord;
vec2 vel = texture2D(u_vel, uv) 等; // 需获取该点速度
vec2 back_pos = uv - dt * vel / cellSize;
vec2 new_vel = texture2D(u_vel, back_pos);
// Level Set 同样反向追踪
float new_phi = texture2D(levelset, back_pos);
输出到 tmp_vel 和 tmp_phi。

Pass 3: 表面张力计算
输入：levelset（已重初始化）

输出：curvature 纹理（单通道）或直接计算张力力

步骤：

计算法向 n = normalize(∇φ)（需差分 ∇φ）。
计算曲率 κ = div(n)（需二阶差分）。
计算光滑界面指示 delta = smoothDelta(φ)。
输出张力体积力分量 f_st_x, f_st_y 存储到临时纹理（或直接加到速度上）。
着色器采用 Sobel 或中心差分计算偏导。

Pass 4: 外力和粘性
输入：tmp_vel（对流后），curvature，levelset

输出：tmp_vel2

显式更新：

glsl
vec2 vel = texture2D(tmp_vel, uv);
float phi = texture2D(levelset, uv);
// 重力
vel.y -= gravity * dt;
// 表面张力 (只在水区域 phi>0 附近)
float st = sigma * kappa * delta(phi);
vel.x += st * nx * dt;
vel.y += st * ny * dt;
// 粘性
vel += nu * dt * laplacian(vel);
粘性拉普拉斯需要读取邻域速度，可单独写一个 pass，或这里合并。

Pass 5: 压力投影（最关键）
输入：tmp_vel2，levelset

输出：pressure，new_vel

算法：

计算散度 div = (dU/dx + dV/dy)（中心差分）。
迭代求解泊松方程 ∇·∇p = div / dt。
使用 Jacobi 迭代（最容易并行）：
glsl
p_new = (p_left + p_right + p_up + p_down - h^2 * div) / 4.0;
迭代 20~40 次，每次读取上一步的 pressure 纹理，输出到双缓冲切换。
修正速度：U_new = U_tmp - dt * (dp/dx)，同样差分。
注意自由表面边界：在空气区域（phi < 0）强制 p = 0，在迭代中直接赋值。

Pass 6: 飞溅检测（可选，但建议）
输入：levelset，new_vel

输出：飞溅数据（通过 CPU 回读或单独粒子纹理）

在 GPU 上检测孤立水团：用连通组件标记算法（并查集在 GPU 上复杂），通常可以每 N 帧在 CPU 端做一次连通分析，性能可接受。

🧩 着色器编写的关键技巧
1. 纹理边界处理
使用 gl_FragCoord 或纹理坐标，采样时使用 texture2D 的 clamp 或 repeat 模式。

固体边界直接修改值：在 Pass 0 中通过判断坐标是否在子弹形状内来强制速度为零。

2. 双缓冲乒乓切换
对于需要迭代的步骤（如压力 Jacobi），准备两个纹理 pressureA 和 pressureB，每步交换。

3. 导数与拉普拉斯模板
glsl
float dudx = (uv.右 - uv.左) / (2.0 * dx);
float laplacian_u = (u_right + u_left + u_up + u_down - 4.0*u_center) / (dx*dx);
4. 优化：合并简单 Pass
例如，重初始化可以紧接着 Level Set 对流之后，不需要存储中间纹理，如果迭代次数不多可合并。

💻 与 Three.js 集成方式
使用 GPUComputationRenderer 管理计算纹理和 shader material。

每个 Pass 对应一个 Mesh 渲染到一个 WebGLRenderTarget，使用 OrthographicCamera + PlaneGeometry。

或使用 WebGPUComputeRenderer（Three.js 已支持部分）。

✅ 你的代码需要达到的“着色器级别”的具体产出
至少 6~8 个独立的着色器代码块（Fragment Shader 或 Compute Shader）。

每个着色器只做一件事（如对流、重初始化、曲率、压力迭代）。

数据流完全通过纹理传递，无 CPU 同步（除最终显示）。

能够展示一个简单的测试场景：水滴下落撞击三角形，产生飞溅。
