我们逐步细化“添加页面管理器”和“对现有代码进行解耦”的具体实施步骤。目标是将当前单体 App.tsx 拆分为：全局状态管理器 + 页面路由 + 3D 游戏核心。

一、当前项目现状（需要解耦的部分）
从你提供的文档可知，App.tsx 目前包含：

3D Canvas 及内部组件（TerrainRenderer、PaperCharacter、SceneSetup 等）

MovementController（处理键盘移动、相机控制、射击管理器、子弹池等）

UI 组件（StatusPanel、DialogBubble、LoadingIndicator、Crosshair 等）

大量 useState/useRef（鼠标位置、射击状态、子弹数组、锁定状态等）

这些混杂在一起，导致：

无法轻易添加“菜单”“商店”等独立页面

战斗逻辑和 UI 强耦合

五、注意事项
不要一次性移动所有代码：分步进行，每步测试。

保持游戏可运行：移动一部分后立即验证移动、射击等功能是否正常。

敌人 AI：可以先实现一个简单的 EnemyManager，在战斗开始后生成几个静态靶子，并将它们传入射击系统。

1.2 创建 src/core/eventBus.ts
typescript
type EventCallback = (...args: any[]) => void;

class EventBus {
  private events: Map<string, EventCallback[]> = new Map();

  on(event: string, callback: EventCallback) {
    if (!this.events.has(event)) this.events.set(event, []);
    this.events.get(event)!.push(callback);
  }

  off(event: string, callback: EventCallback) {
    const callbacks = this.events.get(event);
    if (callbacks) {
      const index = callbacks.indexOf(callback);
      if (index !== -1) callbacks.splice(index, 1);
    }
  }

  emit(event: string, ...args: any[]) {
    const callbacks = this.events.get(event);
    if (callbacks) callbacks.forEach(cb => cb(...args));
  }
}

export const eventBus = new EventBus();
